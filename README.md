# README

House price prediction model


Overview of the project:

  This project is a house price prediction application powered by machine learning, designed to estimate property values based on user-provided inputs. The system utilizes a Random Forest Regressor trained on sample data that includes features such as the number of bedrooms, bathrooms, total area, and location. To enhance model performance, categorical data like location is encoded, and all features are normalized using StandardScaler.Once trained, the model and scaler are saved using joblib, allowing the system to generate predictions without retraining each time. The application also integrates a SQLite database, where all prediction results along with their corresponding input details are stored in a table named predictions. Users interact with the system through a menu-driven command-line interface, where they can choose to make a new prediction, view previously stored results, or exit the program. During prediction, the system accepts input, processes it, calculates the estimated price, displays the result, and records it in the database. When accessing the history feature, all saved entries are fetched and presented to the user. Overall, this project demonstrates the integration of machine learning, database management, and user interaction in a single, efficient, and user-friendly application.

  Features:
    The system builds a machine learning model using a Random Forest Regressor to predict housing prices based on key features such as bedrooms, bathrooms, property size, and location. Since location is categorical, it is first converted into numerical form, and all input features are standardized using StandardScaler to ensure better prediction performance. After training, both the model and scaler are stored as .pkl files using joblib, allowing the application to reuse them for future predictions without the need for retraining. Alongside this, the program initializes a SQLite database containing a table named predictions, where each prediction along with its input parameters is recorded. The application operates through a command-line menu interface, giving users the option to generate a new prediction, access previously saved results, or exit the program. When a prediction is requested, the system gathers user input, processes it through the trained model, displays the estimated price, and stores the result in the database. Users can also retrieve and review all past predictions directly from the database. The program runs continuously in a loop until the user decides to exit, ensuring a smooth and interactive experience.

  Steps to install and Run the Project:
    First, make sure that Python is installed on your system. Next, install the required Python libraries by running the command pip install pandas scikit-learn joblib in your terminal or command prompt, because these packages are needed for data handling and machine-learning operations. After that, save the provided code into a Python file, such as house_price_system.py, and keep it in a folder where you want the project to run. There is no need to manually create the database because the program will automatically generate the SQLite database file named house_data.db when it runs for the first time.Once everything is set up, open a terminal, navigate to the folder containing the file, and run the project using the command python house_price_system.py. The program will train the model, save the required files, and then display a menu on the screen. You can follow the on-screen instructions to predict house prices, view prediction history, or exit the application. The project will continue running until you choose to close it.

Instructions for Testing:
  To begin testing the project, first run the program using the command python house_price_system.py in your terminal after installing the required dependencies. When the menu appears, select the option to predict a house price by entering 1. The program will then ask for the number of bedrooms, bathrooms, the area in square feet, and the location. You can enter sample values such as three bedrooms, two bathrooms, 1400 square feet, and “city” to check whether the prediction is generated correctly. After submitting the details, the program should display a predicted price and confirm that the result has been saved to the database.

Next, return to the main menu and choose the second option to view prediction history. This will allow you to verify that the previously generated prediction has been stored successfully in the database. You can repeat the process with different inputs, including suburb or rural locations, to ensure that the model handles all cases properly. It is also useful to test invalid inputs, such as typing letters instead of numbers or entering an unsupported location, to confirm that the program responds appropriately.

Finally, you can close the application by selecting the exit option from the menu. After exiting, you may also check that the house_data.db file and the saved model files have been created in the project folder to confirm that the system is functioning as expected.
